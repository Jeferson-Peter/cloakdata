## Highlights

- Added a new set of practical anonymization and sanitization methods for structured and free-text data
- Expanded time-based anonymization with richer `coarsen_datetime` modes
- Added runnable examples and tests for every new built-in method
- Added a demo dataset and config for manual parquet-based experimentation

## Features

- New replacement methods:
  - `hash_value`
  - `redact_regex`
  - `replace_with_hash_bucket`
- New masking method:
  - `mask_credit_card`
- New randomization method:
  - `noise_numeric`
- New generalization methods:
  - `top_k_bucket`
  - `generalize_zip_code`
  - `coarsen_datetime`
- New numeric transform:
  - `clip_range`
- New utility method:
  - `null_if_matches`

## Time-Based Anonymization

- `coarsen_datetime` now supports:
  - `bucket`
  - `minute_of_day_bucket`
  - `hour`
  - `part_of_day`
  - `weekday`
  - `weekend_weekday`
  - `business_hours`
- `business_hours` can now be configured with:
  - `start_hour`
  - `end_hour`
  - `include_weekends`
- `minute_of_day_bucket` supports both:
  - time-only output
  - full datetime output

## Quality Improvements

- Added focused tests for all new methods and temporal modes
- Added runnable examples for each new built-in method added in this release
- Expanded README coverage for new methods and behavior notes
- Added a demo folder with a sample parquet file and anonymization config
- Added a roadmap document for future method expansion

## Upgrade Notes

- This release is additive and does not introduce a new public API break
- `hash_value` is the right choice for one-way stable pseudonymization
- `replace_with_hash_bucket` is deterministic bucketing and can intentionally create collisions
- `coarsen_datetime` remains the recommended temporal abstraction method for timestamp-level anonymization
