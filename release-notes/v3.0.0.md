## Highlights

- Reorganized built-in anonymization methods into domain modules under `native_methods`
- Added automatic built-in registration with `@native_method`
- Removed the legacy `AnonymizationMethods` and `cloakdata.methods` API surface
- Improved validation, functional coverage, and documentation

## Breaking Changes

- `AnonymizationMethods` is no longer part of the public API
- `cloakdata.methods` was removed
- Built-in extension inside the library now uses plain functions plus `@native_method`

## Features

- Built-in methods are now organized by domain:
  - masking
  - generalize
  - rounding
  - replace
  - randomize
  - sequential
  - utils
- New automatic built-in method registration via `@native_method`
- Explicit native method catalog for the registry
- Functional test coverage for end-to-end anonymization scenarios

## Quality Improvements

- Refactored `mask_partial` and `round_date` to use native Polars expressions
- Fixed `validate()` to use the active dispatch map correctly
- Updated documentation and contribution guidance for the new architecture
- Improved release workflow alignment with automated PyPI publishing and GitHub Releases

## Upgrade Notes

- Runtime custom methods continue to work with `register_method(...)`
- If you maintained built-in methods inside the library, migrate them to `src/cloakdata/native_methods/`
- Register new built-in methods with `@native_method`

## Install

```bash
pip install -U cloakdata
```

## Distribution

This release is published to PyPI. GitHub Release assets are not used as the distribution channel.
